API Reference

v2.1

The Globex Sky REST API lets you integrate our sourcing platform into your own applications. All requests and responses use JSON.

Introduction

The Globex Sky API is organized around REST principles. Our API uses standard HTTP response codes, authentication, and verbs. All API requests must be made over HTTPS.

Base URL https://api.globexsky.com/v2
Note: All API responses include a X-Request-ID header for debugging. Include this in any support requests.

Authentication

The Globex Sky API uses API keys to authenticate requests. You can view and manage your API keys in the Developer Dashboard.

Pass your API key as a bearer token in the Authorization header:

Header Authorization: Bearer YOUR_API_KEY
Security: Never expose your API key in client-side code. All requests should be proxied through your backend server.

Code Examples

# List products with authentication
curl -X GET "https://api.globexsky.com/v2/products" \
  -H "Authorization: Bearer sk_live_xxxxxxxxxxxx" \
  -H "Content-Type: application/json"
import requests

API_KEY = "sk_live_xxxxxxxxxxxx"
BASE_URL = "https://api.globexsky.com/v2"

headers = {
    "Authorization": f"Bearer {API_KEY}",
    "Content-Type": "application/json"
}

response = requests.get(f"{BASE_URL}/products", headers=headers)
products = response.json()
print(products)
const API_KEY = 'sk_live_xxxxxxxxxxxx';
const BASE_URL = 'https://api.globexsky.com/v2';

async function fetchProducts() {
  const response = await fetch(`${BASE_URL}/products`, {
    headers: {
      'Authorization': `Bearer ${API_KEY}`,
      'Content-Type': 'application/json'
    }
  });
  const data = await response.json();
  console.log(data);
}

package main

    "fmt"
    "net/http"
    "io/ioutil"
)

func main() {
    client := &http.Client{}
    req, _ := http.NewRequest("GET",
        "https://api.globexsky.com/v2/products", nil)
    req.Header.Set("Authorization", "Bearer sk_live_xxxxxxxxxxxx")

Error Codes

The API uses standard HTTP status codes. Errors include a JSON body with details.

CodeStatusDescription
400Bad RequestInvalid request parameters or malformed JSON body.
401UnauthorizedMissing or invalid API key.
403ForbiddenAPI key lacks permission for this endpoint.
404Not FoundThe requested resource does not exist.
409ConflictRequest conflicts with existing state (e.g., duplicate SKU).
422Unprocessable EntityValidation failed. Check the errors array in the response.
429Too Many RequestsRate limit exceeded. Check X-RateLimit-Reset header.
500Internal Server ErrorServer-side error. Contact support with X-Request-ID.
